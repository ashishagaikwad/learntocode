HTML HEAD ELEMENTS

Following are head elements

  1. title
  2. base
  3. link
  4. meta
  5. basefont(DEPRECATED)
  6. script
  7. style

base:

The base element, which is contained in the head of the document, provides a method for defining the base URL for all links and form submissions on a page. It also provides a common target (in the form of a named window) for all of these links or form submissions.

It has two attributes:

  1. href: specifies the base URL for all relative URLs in the document
  2. target: specifies the default target window in which to display content from links in the current document

link:

The link element is used to create links between the referencing document and an external resource

It has following attributes:

  1. charset : The charset attribute is intended to identify the character set used in the document that’s referenced within the link element.
  2. href : The href defines the location of the resource
  3. media : If your page is to be viewed on a variety of media, you may want to apply to it a number of separate styles that are specific to each medium.These subtle changes are made possible by the media attribute.Example : This example shows the application of two different styles for two different media:
  4. rel : The rel attribute defines the relationship that the linked resource has to the document from which it’s referenced
  5. rev : NOT SUPPORTED IN ANY BROWSERS.It defines the relationship that the referenced document would classify itself as having with this document. Not supported in any browsers.
  6. target : NOT SUPPORTED IN ANY BROWSERS.The target attribute is used in the link element to identify the window or frame—which may be in a frameset, an iframe, or even a popup window with a custom window name—that the resource referenced in the href attribute applies to.
  7. type : NOT TO BE USED IN HTML 5. type is an optional attribute of the link element, that is used to identify the MIME type of the linked resource.

meta :

The meta element provides information about the following document content; that information may be used by the user agent (that is, the browser) to decide how to render content, or it may be meta information that’s provided for indexing purposes

  1. content : content is a required attribute. It’s this part of the meta element that contains the actual meta information, which is defined by the name attribute
  2. http-equiv: You can apply many settings to a web server in the form of HTTP response headers—information that’s sent back with the response to any request made for a resource on that server. However, not everyone has direct access to their web site’s server configuration options; if this is the case, you can use the meta element to simulate an HTTP response header
  3. name : he name attribute is the counterpart to the content attribute, and simply provides a name for the information included in the content attribute.
